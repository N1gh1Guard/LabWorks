## 2.1. Алгоритмы сортировки: классификация

Алгоритмы сортировки можно классифицировать по **устойчивости**, по **использованию памяти** (in-place или нет), по **принципу работы** (обменные, выбором, вставками, слиянием, распределительные) и по **типу сравнений** (сравнительные — только сравнение ключей; непременно сравнительные — подсчёт, поразрядная).

---

**Устойчивость (stable sort).** Сортировка **устойчива**, если элементы с равными ключами сохраняют относительный порядок после сортировки. То есть если до сортировки элемент A шёл раньше элемента B и ключи A и B равны, то после сортировки A по-прежнему раньше B. Устойчивость нужна, когда сортируем по одному полю, но важен порядок по другому (например, сначала сортировка по фамилии, потом по имени — при равных фамилиях порядок имён сохраняется). Пример: пузырьковая и сортировка вставками при сравнении «строго меньше» — устойчивы; быстрая сортировка и сортировка выбором при обмене элементов — неустойчивы.

**In-place.** Алгоритм сортирует **in-place**, если кроме самого массива (и возможно O(1) переменных: индексы, стек рекурсии при ограниченной глубине) не использует память, растущую с n. То есть дополнительная память O(1). Сортировка слиянием обычно требует буфер размера O(n) для слияния двух половин — не in-place (если не использовать сложные варианты с O(1) доп. памяти). Быстрая сортировка в типичной реализации — in-place: только стек вызовов O(log n) в среднем; при худшем разбиении стек O(n).

**Принципы работы.**

- **Обменные:** на каждом шаге сравниваются пара элементов и при необходимости они **обмениваются** местами. Примеры: пузырьковая сортировка (сравниваем соседей, меняем местами), быстрая сортировка (разделение относительно pivot с обменами).
- **Выбором:** на каждом шаге среди неотсортированной части выбирается **экстремум** (минимум или максимум) и ставится на своё место. Примеры: сортировка выбором (минимум в начало), пирамидальная сортировка (куча для быстрого выбора максимума).
- **Вставками:** массив мысленно делится на **отсортированную** часть и неотсортированную; очередной элемент **вставляется** в нужное место отсортированной части. Примеры: сортировка вставками, сортировка Шелла (вставки с убывающим шагом).
- **Слиянием (merge):** массив рекурсивно делится пополам, каждая половина сортируется, затем две отсортированные половины **сливаются** в один отсортированный массив. Доп. память под слияние — O(n). Пример: сортировка слиянием.
- **Распределительные:** порядок достигается не сравнением пар элементов, а **распределением** по корзинам или по разрядам. Примеры: сортировка подсчётом (count), поразрядная (radix), блочная (bucket). Могут давать O(n) при ограничениях на ключи.

**Сравнительные и непременно сравнительные.** Большинство алгоритмов выше — **сравнительные**: порядок определяется только результатами сравнений «меньше / больше». Для них нижняя оценка сложности Ω(n log n) при сравнении в среднем. **Непременно сравнительные** (подсчёт, radix) используют дополнительную информацию о ключах (диапазон, разряды) и могут достигать O(n).

## 2.2. Обменные сортировки: общий принцип, примеры, асимптотическая оценка сложности

**Принцип.** Обменные (сравнительно-обменные) сортировки упорядочивают массив, **сравнивая пары элементов** и **обменивая** их местами, если порядок нарушен. То есть базовая операция — сравнение двух элементов и при необходимости `swap(a[i], a[j])`. Порядок достигается многократным проходом по массиву или по подмассивам и обменами. К обменным относят **пузырьковую сортировку**, **шейкерную (коктейльную)** и **быструю сортировку** (Quick Sort). Устойчивость и сложность зависят от того, какие пары сравниваются и как выбирается опорный элемент.

**Общая идея.** Пузырьковая и шейкерная — многократные проходы по соседним парам, обмен при «неправильном» порядке; быстрая — выбор опорного элемента (pivot), разбиение массива обменами так, что слева от границы элементы меньше pivot, справа — не меньше, затем рекурсия по частям.

---

**1. Пузырьковая сортировка (Bubble Sort).** На каждом проходе сравниваем соседние элементы `a[j]` и `a[j+1]`; если `a[j] > a[j+1]`, меняем их местами. За один проход «всплывает» максимум в конец; после n проходов массив отсортирован. Устойчива (при сравнении «строго больше» равные не обмениваются). Сложность: O(n²) сравнений и обменов в худшем и среднем случае; в лучшем (уже отсортирован) можно остановиться при проходе без обменов — O(n).

**Код — пузырьковая сортировка:**
```cpp
#include <algorithm>
#include <iterator>

template<typename RandomIt>
void bubble_sort(RandomIt first, RandomIt last) {
    if (first == last) return;
    auto n = std::distance(first, last);
    for (decltype(n) i = 0; i < n; ++i) {
        bool swapped = false;
        for (decltype(n) j = 0; j < n - 1 - i; ++j) {
            auto it = first; std::advance(it, j);
            auto next = it; ++next;
            if (*next < *it) {
                std::iter_swap(it, next);
                swapped = true;
            }
        }
        if (!swapped) break;  // уже отсортировано — выход
    }
}

// Вариант для массива int[]
void bubble_sort_int(int* a, size_t n) {
    for (size_t i = 0; i < n; ++i)
        for (size_t j = 0; j < n - 1 - i; ++j)
            if (a[j] > a[j+1]) std::swap(a[j], a[j+1]);
}
```

---

**2. Шейкерная сортировка (Cocktail / Shaker Sort).** Вариант пузырьковой: проходы по очереди слева направо и справа налево. Слева направо «всплывает» максимум вправо, справа налево «тонет» минимум влево. За один цикл (туда-обратно) уменьшается неотсортированный диапазон с обоих концов. Устойчива. Сложность O(n²); на почти отсортированных может быть меньше обменов, чем у обычной пузырьковой.

**Код — шейкерная сортировка:**
```cpp
template<typename RandomIt>
void shaker_sort(RandomIt first, RandomIt last) {
    if (first == last) return;
    auto left = first, right = last;
    --right;
    while (left < right) {
        for (auto it = left; it != right; ++it) {
            auto next = it; ++next;
            if (*next < *it) std::iter_swap(it, next);
        }
        --right;
        for (auto it = right; it != left; --it) {
            auto prev = it; --prev;
            if (*it < *prev) std::iter_swap(it, prev);
        }
        ++left;
    }
}

void shaker_sort_int(int* a, size_t n) {
    size_t left = 0, right = n - 1;
    while (left < right) {
        for (size_t j = left; j < right; ++j)
            if (a[j] > a[j+1]) std::swap(a[j], a[j+1]);
        --right;
        for (size_t j = right; j > left; --j)
            if (a[j-1] > a[j]) std::swap(a[j-1], a[j]);
        ++left;
    }
}
```

**3. Быстрая сортировка (Quick Sort).** Выбираем **опорный элемент** (pivot), например средний или случайный. **Разбиение (partition):** обменами располагаем элементы так, что слева от границы все строго меньше pivot, справа — не меньше. Граница (индекс) возвращаем. Рекурсивно сортируем левую и правую части. Базовый случай: размер отрезка 0 или 1 — уже отсортирован. Сложность: в среднем при разбиении «примерно пополам» глубина рекурсии O(log n), работа на каждом уровне O(n) → **O(n log n)**. В худшем случае (pivot каждый раз минимум или максимум) глубина O(n) → **O(n²)**. Неустойчива (обмены меняют порядок равных).

**Код — быстрая сортировка (partition + рекурсия):**
```cpp
#include <algorithm>
#include <iterator>

template<typename RandomIt>
RandomIt partition(RandomIt first, RandomIt last, RandomIt pivot_it) {
    using T = typename std::iterator_traits<RandomIt>::value_type;
    T pivot = *pivot_it;
    std::iter_swap(pivot_it, last - 1);  // pivot в конец
    auto i = first;
    for (auto j = first; j != last - 1; ++j)
        if (*j < pivot) std::iter_swap(i++, j);
    std::iter_swap(i, last - 1);
    return i;
}

template<typename RandomIt>
void quick_sort(RandomIt first, RandomIt last) {
    if (last - first <= 1) return;
    auto mid = first + (last - first) / 2;
    auto p = partition(first, last, mid);
    quick_sort(first, p);
    quick_sort(p + 1, last);
}

// Вариант для массива int[] с индексами (разбиение Хоара)
int partition_hoare(int* a, int lo, int hi) {
    int pivot = a[(lo + hi) / 2];
    int i = lo - 1, j = hi + 1;
    while (true) {
        do ++i; while (a[i] < pivot);
        do --j; while (a[j] > pivot);
        if (i >= j) return j;
        std::swap(a[i], a[j]);
    }
}
void quick_sort_int(int* a, int lo, int hi) {
    if (lo >= hi) return;
    int p = partition_hoare(a, lo, hi);
    quick_sort_int(a, lo, p);
    quick_sort_int(a, p + 1, hi);
}
```
Partition в варианте с pivot в конце — O(n). Рекурсия: QuickSort(lo, p), QuickSort(p+1, hi). Улучшения: median-of-three для выбора pivot, случайный pivot, при малом размере подмассива вызывать insertion sort.

## 2.3. Сортировки выбором: общий принцип, примеры, асимптотическая оценка сложности

**Принцип.** Сортировки **выбором** упорядочивают массив так: на каждом шаге среди **неотсортированной** части выбирается **экстремум** (минимум при сортировке по возрастанию или максимум при сортировке по убыванию) и ставится на своё место — в начало или в конец неотсортированной части. То есть базовая операция — поиск минимума (или максимума) в подмассиве и обмен с элементом на текущей позиции. К сортировкам выбором относят **сортировку выбором** (линейный поиск минимума) и **пирамидальную сортировку** (Heap Sort), где максимум извлекается из кучи за O(log n).

**Общая идея.** Сортировка выбором: для каждой позиции i находим минимум среди a[i..n-1] и меняем с a[i]; после n шагов массив отсортирован. Heap Sort: строим max-кучу из массива за O(n), затем n раз забираем максимум (корень кучи), ставим его в конец «отсортированной» части, восстанавливаем кучу siftDown — итого O(n log n).

---

**1. Сортировка выбором (Selection Sort).** На каждом шаге i (от 0 до n-1) ищем **минимум** среди элементов a[i], a[i+1], …, a[n-1] и меняем его местами с a[i]. После шага i префикс a[0..i] отсортирован и содержит i+1 наименьших элементов. Всего n шагов; на шаге i поиск минимума — O(n-i) сравнений, один обмен. Суммарно O(n²) сравнений, O(n) обменов. **Неустойчива:** при выборе минимума равные элементы могут менять относительный порядок (например, если минимум далеко справа). Лучший, средний и худший случай — O(n²); число сравнений не зависит от порядка элементов.

**Код — сортировка выбором:**
```cpp
#include <algorithm>
#include <iterator>

template<typename RandomIt>
void selection_sort(RandomIt first, RandomIt last) {
    if (first == last) return;
    for (auto i = first; i != last; ++i) {
        auto min_it = i;
        for (auto j = std::next(i); j != last; ++j)
            if (*j < *min_it) min_it = j;
        std::iter_swap(i, min_it);
    }
}

void selection_sort_int(int* a, size_t n) {
    for (size_t i = 0; i < n; ++i) {
        size_t minIdx = i;
        for (size_t j = i + 1; j < n; ++j)
            if (a[j] < a[minIdx]) minIdx = j;
        std::swap(a[i], a[minIdx]);
    }
}
```

**2. Пирамидальная сортировка (Heap Sort).** Массив интерпретируется как **двоичная куча** (max-heap): для индекса i родитель (i-1)/2, дети 2i+1 и 2i+2; в корне (индекс 0) — максимум. **Фаза 1 — построение кучи (heapify):** вызываем siftDown для всех внутренних узлов от (n/2)-1 до 0 (снизу вверх). Каждый siftDown опускает узел вниз, пока инвариант кучи не выполнится. Суммарная стоимость heapify — O(n). **Фаза 2 — извлечение максимума n раз:** на каждой итерации корень (максимум) меняем с последним элементом «текущей» кучи, уменьшаем размер кучи на 1, вызываем siftDown от нового корня. Извлечённые максимумы образуют отсортированную часть в конце массива (по возрастанию — если кладём максимум в конец и восстанавливаем max-heap). Итого: O(n) + n·O(log n) = **O(n log n)**. **Неустойчива.** Лучший, средний и худший случай — O(n log n).

**Код — пирамидальная сортировка (Heap Sort):**
```cpp
#include <algorithm>
#include <iterator>

template<typename RandomIt>
void sift_down(RandomIt first, RandomIt last, RandomIt start) {
    auto n = std::distance(first, last);
    auto i = std::distance(first, start);
    while (true) {
        auto left = 2 * i + 1, right = 2 * i + 2;
        auto largest = i;
        if (left < n) {
            auto it_left = first; std::advance(it_left, left);
            if (*it_left > *start) largest = left;
        }
        if (right < n) {
            auto it_right = first; std::advance(it_right, right);
            auto it_largest = first; std::advance(it_largest, largest);
            if (*it_right > *it_largest) largest = right;
        }
        if (largest == i) break;
        auto it_largest = first; std::advance(it_largest, largest);
        std::iter_swap(start, it_largest);
        start = it_largest;
        i = largest;
    }
}

template<typename RandomIt>
void heap_sort(RandomIt first, RandomIt last) {
    if (first == last) return;
    auto n = std::distance(first, last);
    // Построение max-heap (heapify)
    for (auto i = n / 2 - 1; i >= 0; --i) {
        auto it = first; std::advance(it, i);
        sift_down(first, last, it);
    }
    // Извлечение максимума n раз
    for (auto len = n; len > 1; --len) {
        auto last_it = first; std::advance(last_it, len - 1);
        std::iter_swap(first, last_it);
        sift_down(first, first + (len - 1), first);
    }
}

// Вариант для массива int[] (индексы 0..n-1)
void sift_down_int(int* a, int n, int i) {
    while (true) {
        int left = 2 * i + 1, right = 2 * i + 2, largest = i;
        if (left < n && a[left] > a[largest]) largest = left;
        if (right < n && a[right] > a[largest]) largest = right;
        if (largest == i) break;
        std::swap(a[i], a[largest]);
        i = largest;
    }
}
void heap_sort_int(int* a, int n) {
    for (int i = n / 2 - 1; i >= 0; --i)
        sift_down_int(a, n, i);
    for (int len = n; len > 1; --len) {
        std::swap(a[0], a[len - 1]);
        sift_down_int(a, len - 1, 0);
    }
}
```
Heapify: вызов siftDown для индексов от (n/2)-1 до 0. Извлечение: обмен a[0] с a[len-1], siftDown(0) для кучи размера len-1. Итог — массив отсортирован по возрастанию.

## 2.4. Сортировки вставкой: общий принцип, примеры, асимптотическая оценка сложности

**Принцип.** Сортировки **вставками** упорядочивают массив так: массив мысленно делится на **отсортированную** часть (обычно префикс a[0..i-1]) и **неотсортированную** (a[i..n-1]). Очередной элемент a[i] **вставляется** в нужное место отсортированной части: сдвигаем вправо все большие элементы и ставим a[i] в освободившуюся позицию. То есть базовая операция — поиск позиции вставки в отсортированной части и сдвиг элементов. К сортировкам вставками относят **сортировку вставками** (вставка в префикс с шагом 1) и **сортировку Шелла** (вставки с убывающим шагом gap).

**Общая идея.** Сортировка вставками: для каждого i от 1 до n-1 вставляем a[i] в уже отсортированный префикс a[0..i-1] (линейный поиск места и сдвиг). Shell: выполняем вставки с шагом gap (например n/2, n/4, …, 1); при большом gap элементы «быстрее» попадают ближе к нужному месту, затем при gap=1 массив почти отсортирован — мало сдвигов.

---

**1. Сортировка вставками (Insertion Sort).** На каждом шаге i (от 1 до n-1) элемент a[i] вставляем в отсортированный префикс a[0..i-1]. Сохраняем a[i] во временную переменную x; сдвигаем вправо все элементы a[j-1] > x (j от i вниз до 1); ставим x в позицию j. Сравнений и сдвигов на шаге i в худшем случае — до i; суммарно O(n²). В **лучшем** случае (массив уже отсортирован) на каждом шаге одно сравнение, сдвигов нет — O(n). На **почти отсортированных** данных — близко к O(n). **Устойчива:** при сравнении «строго больше» равные не сдвигаются, порядок сохраняется. Часто используют для малых массивов (n ≤ 10–50) или как финальный проход после быстрой сортировки по подмассивам.

**Код — сортировка вставками:**
```cpp
#include <algorithm>
#include <iterator>

template<typename RandomIt>
void insertion_sort(RandomIt first, RandomIt last) {
    if (first == last) return;
    for (auto i = std::next(first); i != last; ++i) {
        auto value = *i;
        auto j = i;
        while (j != first) {
            auto prev = std::prev(j);
            if (!(*prev > value)) break;
            *j = *prev;
            j = prev;
        }
        *j = value;
    }
}

void insertion_sort_int(int* a, size_t n) {
    for (size_t i = 1; i < n; ++i) {
        int x = a[i];
        size_t j = i;
        while (j > 0 && a[j - 1] > x) {
            a[j] = a[j - 1];
            --j;
        }
        a[j] = x;
    }
}
```

**2. Сортировка Шелла (Shell Sort).** Идея: выполнять **вставки с шагом gap** вместо 1. При gap > 1 сортируем подпоследовательности a[0], a[gap], a[2*gap], …; a[1], a[1+gap], …; и т.д. — в каждой подпоследовательности применяем сортировку вставками. Затем уменьшаем gap (например, gap = gap/2) и повторяем, пока gap не станет 1. При gap=1 — обычная сортировка вставками по всему массиву; к этому моменту массив уже «почти» отсортирован благодаря предыдущим проходам, поэтому сдвигов мало. **Последовательность шагов:** классическая — n/2, n/4, …, 1; или последовательность Кнута 1, 4, 13, 40, … (3*h+1). Сложность зависит от выбора шагов: при n/2, n/4, … в худшем случае O(n²); при последовательности Кнута — O(n^1.5) или лучше (практически около O(n^1.25)–O(n^1.3)). **Неустойчива:** обмены с шагом gap меняют относительный порядок равных элементов.

**Код — сортировка Шелла:**
```cpp
#include <algorithm>
#include <iterator>

template<typename RandomIt>
void shell_sort(RandomIt first, RandomIt last) {
    auto n = std::distance(first, last);
    if (n <= 1) return;
    // Последовательность шагов: n/2, n/4, ..., 1
    for (auto gap = n / 2; gap >= 1; gap /= 2) {
        for (auto i = first + gap; i != last; ++i) {
            auto value = *i;
            auto j = i;
            while (j - first >= gap) {
                auto prev = j - gap;
                if (!(*prev > value)) break;
                *j = *prev;
                j = prev;
            }
            *j = value;
        }
    }
}

// Вариант с последовательностью Кнута (1, 4, 13, 40, ...)
template<typename RandomIt>
void shell_sort_knuth(RandomIt first, RandomIt last) {
    auto n = std::distance(first, last);
    if (n <= 1) return;
    decltype(n) gap = 1;
    while (gap < n / 3) gap = 3 * gap + 1;  // максимальный шаг Кнута < n/3
    for (; gap >= 1; gap = (gap - 1) / 3) {
        for (auto i = first + gap; i != last; ++i) {
            auto value = *i;
            auto j = i;
            while (j - first >= gap) {
                auto prev = j - gap;
                if (!(*prev > value)) break;
                *j = *prev;
                j = prev;
            }
            *j = value;
        }
    }
}

void shell_sort_int(int* a, size_t n) {
    for (size_t gap = n / 2; gap >= 1; gap /= 2)
        for (size_t i = gap; i < n; ++i) {
            int x = a[i];
            size_t j = i;
            while (j >= gap && a[j - gap] > x) {
                a[j] = a[j - gap];
                j -= gap;
            }
            a[j] = x;
        }
}
```
При gap=1 внутренний цикл совпадает с сортировкой вставками. Последовательность Кнута даёт лучшую оценку на практике.

## 2.5. Декомпозиция задач на примере алгоритма быстрой сортировки

**Что такое декомпозиция.** Декомпозиция — это **разбиение задачи на подзадачи того же типа**, решаемые независимо (или почти независимо), с последующим объединением результатов или завершением, когда подзадача становится тривиальной. Типичная схема: **базовый случай** (подзадача настолько мала, что решается сразу) и **рекурсивный случай** (разбиваем задачу на части, решаем каждую часть тем же алгоритмом, объединяем). Примеры: бинарный поиск (делим диапазон пополам), сортировка слиянием (делим массив пополам, сортируем половины, сливаем), **быстрая сортировка** (разделяем массив относительно pivot, рекурсивно сортируем левую и правую части).

**Быстрая сортировка как декомпозиция.** Задача: отсортировать отрезок массива a[lo..hi]. Декомпозиция:

1. **Базовый случай:** если размер отрезка 0 или 1 (lo ≥ hi или lo + 1 = hi), отрезок уже отсортирован — выходим.
2. **Разделение (partition):** выбираем опорный элемент pivot (например, a[(lo+hi)/2]). Переставляем элементы так, что слева от некоторой границы p все элементы строго меньше pivot, справа — не меньше. Сложность разделения — O(n), где n = hi − lo + 1.
3. **Рекурсия:** вызываем QuickSort для левой части a[lo..p] и для правой части a[p+1..hi]. Объединять ничего не нужно: после partition слева все меньше pivot, справа не меньше, pivot на границе; после сортировки обеих частей весь отрезок отсортирован.

Итог: задача «отсортировать n элементов» сводится к двум задачам «отсортировать k элементов» и «отсортировать n−k−1 элементов» плюс O(n) работы на partition. Размеры подзадач зависят от выбора pivot и от того, как partition разбивает массив.

**Почему в среднем O(n log n).** При **хорошем разбиении** (размеры обеих частей примерно n/2) глубина рекурсии — O(log n): на каждом уровне рекурсии суммарный размер всех подзадач не больше n, работа на уровне — O(n) (partition для каждого подмассива). Итого: глубина O(log n) × работа на уровень O(n) = **O(n log n)**. В среднем при случайном выборе pivot разбиение «примерно пополам» получается часто, поэтому средняя сложность O(n log n).

**Почему в худшем случае O(n²).** При **плохом разбиении** (одна часть пустая или почти пустая, другая содержит почти все элементы) глубина рекурсии становится O(n): на каждом шаге уменьшается только один элемент (pivot). Суммарная работа: n + (n−1) + (n−2) + … = O(n²). Худший случай реализуется, например, когда pivot каждый раз минимум или максимум (уже отсортированный массив при выборе крайнего элемента как pivot).

**Улучшения, снижающие вероятность худшего случая.**

- **Median-of-three:** в качестве pivot брать медиану из a[lo], a[(lo+hi)/2], a[hi]. Тогда на уже отсортированном (и на почти отсортированном) массиве pivot будет средним элементом — разбиение будет ближе к равному.
- **Случайный pivot:** выбирать случайный индекс в [lo, hi]. В среднем разбиение близко к равному; худший случай возможен, но с малой вероятностью.
- **Insertion sort для малых подмассивов:** при размере подмассива меньше порога (например, 10–50) вызывать сортировку вставками вместо рекурсии. Уменьшает число вызовов и накладные расходы; на малых массивах вставками часто быстрее.

**Схема рекурсии (при хорошем разбиении).** Уровень 0: один вызов для n элементов, partition O(n). Уровень 1: два вызова для ~n/2, суммарно partition O(n). Уровень 2: четыре вызова для ~n/4, суммарно O(n). … Уровень log n: подзадачи размера 1, базовый случай. Итого уровней O(log n), на каждом уровне работа O(n) → **O(n log n)**.
