# 1. СТРУКТУРЫ ДАННЫХ

Конспект для подготовки к экзамену по курсу «Информатика (организация и поиск данных)».

---

## 1.1. Классификация структур данных

**По способу размещения в памяти:**
- **Последовательные (континуальные)** — элементы хранятся в смежных ячейках подряд. Адрес i-го элемента: `base + i * sizeof(T)` — линейная адресация. Прямой доступ за O(1): индекс преобразуется в адрес одной арифметической операцией. Недостаток: вставка или удаление в середине требует сдвига всех последующих элементов — O(n). Хорошая кеш-локальность: последовательный обход обращается к смежным адресам.
- **Связанные** — каждый элемент (узел) содержит данные и указатели на соседей. Элементы могут быть разбросаны по памяти. Вставка и удаление при наличии указателя на узел — O(1), так как достаточно обновить связи. Нет прямого доступа по индексу: для i-го элемента нужен обход O(i).
- **Комбинированные** — сочетают оба подхода. Хеш-таблица: массив бакетов (континуально) и цепочки узлов в бакетах при коллизиях (связанные).

**По типу доступа:**
- **Прямой (Random Access)** — доступ к элементу по индексу за O(1). Примеры: массив, динамический массив.
- **Последовательный** — обход только в порядке следования, вперёд или вперёд-назад. Примеры: список, поток.
- **Ассоциативный** — доступ по ключу. Хеш — O(1) в среднем, дерево — O(log n).

**По динамичности:** статические (размер задан при создании) / динамические (размер может расти и уменьшаться).

**По мутабельности:** изменяемые (операции модифицируют структуру in-place) / неизменяемые (каждая операция возвращает новую структуру, исходная не меняется).

## 1.2. Массивы, динамические массивы. Основные операции, асимптотика

**Статический массив** — непрерывный блок памяти фиксированного размера. Синтаксис `arr[i]` эквивалентен `*(arr + i)`: адресная арифметика. Компилятор учитывает sizeof(T), поэтому сдвиг на i элементов, а не байт. Прямой доступ O(1).

**Динамический массив** хранит: указатель на буфер, size (текущее число элементов), capacity (размер выделенной памяти). При push_back: если size < capacity, просто записываем в конец. Если size >= capacity, выделяем новый буфер (обычно в 2 раза больше), копируем все элементы, освобождаем старый. Стоимость одного push_back в худшем случае O(n), но амортизированно O(1).

**Амортизированный анализ (метод банкира):** при каждой вставке «платим» две монеты — одну на саму запись, вторую «откладываем». При удвоении буфера накопленные монеты покрывают копирование: каждый из n элементов при последнем копировании тратит одну монету. Итого n вставок — O(n) работы, т.е. O(1) на операцию в среднем.

**Вставка в начало или середину** — сдвиг всех последующих элементов вправо, O(n). **Удаление** — сдвиг влево, O(n).

| Операция | Оценка |
|----------|--------|
| Доступ по индексу | O(1) |
| push_back | O(1) амортиз. |
| Вставка в начало/середину | O(n) |
| Удаление | O(n) |

## 1.3. Связанные списки и их вариации

**Односвязный список:** узел содержит данные и указатель next на следующий. Храним head (и tail для O(1) push_back). push_front — новый узел в начало, O(1). push_back — в конец при наличии tail, O(1). insert_after(узел) — вставка после заданного узла, O(1). remove_after(узел) — удаление следующего, O(1). Удаление заданного узла при отсутствии указателя на предыдущий требует обхода O(n). Поиск по значению — O(n).

**Двусвязный список:** узел `{data, prev, next}`. При вставке и удалении обновляем связи соседей с обеих сторон — O(1) при указателе на узел. Удаление текущего узла без поиска предыдущего — O(1). Дополнительная память на один указатель на узел.

**Линейный** — последний узел имеет next = nullptr. **Кольцевой** — последний указывает на первый, tail->next = head. В двусвязном кольце также head->prev = tail. Обход — по счётчику или до возврата к началу. Удобно для циклических очередей, round-robin.

## 1.4. Очередь. Способы реализации и оценка асимптотики

**Очередь FIFO** — First In, First Out. Операции: enqueue (добавить в конец), dequeue (извлечь из начала).

**Кольцевой буфер:** массив фиксированного размера, индексы head и tail. При enqueue пишем в tail, tail = (tail+1) % capacity. При dequeue читаем из head, head = (head+1) % capacity. Обе операции O(1). Переполнение — когда (tail+1)%capacity == head. При необходимости расширения — перераспределение с копированием O(n).

**На связном списке:** храним указатели на голову и хвост. enqueue — вставка в конец (после tail), dequeue — удаление из начала. O(1) для обеих операций. Размер не ограничен (если не ограничена память).

---

## 1.5. Стек. Способы реализации и оценка асимптотики

**Стек LIFO** — Last In, First Out. Операции: push (положить наверх), pop (снять сверху), top (посмотреть верхний без извлечения).

**На динамическом массиве:** вершина стека — конец массива. push = push_back, pop = pop_back, top = доступ к последнему элементу. O(1) амортизированно для push, O(1) для pop и top.

**На односвязном списке:** вершина стека — голова списка. push — вставка в начало, pop — удаление из начала. O(1) на все операции. Не требует предварительного выделения памяти под размер.

## 1.6. Двоичная куча. Основные операции. Способы реализации и оценка асимптотики

**Инвариант кучи:** для каждого узла родитель не меньше (max-heap) или не больше (min-heap) обоих детей. Полное бинарное дерево — все уровни заполнены слева направо, кроме, возможно, последнего.

**Хранение в массиве:** обход по уровням (слева направо). Для индекса i: родитель (i-1)/2, левый ребёнок 2i+1, правый 2i+2. Нет указателей, хорошая кеш-локальность.

**siftUp (просеивание вверх):** при вставке ставим новый элемент в конец массива. Пока он нарушает инвариант с родителем — меняем местами с родителем. O(log n) — не более высоты дерева.

**siftDown (просеивание вниз):** при извлечении максимума (min-heap — минимума) ставим последний элемент в корень. Пока он нарушает инвариант с детьми — меняем с большим (или меньшим для min-heap) ребёнком. O(log n).

**Heapify (построение кучи из массива):** вызываем siftDown для всех внутренних узлов от (n/2)-1 до 0 (снизу вверх). Анализ: сумма «высот» узлов даёт O(n), а не O(n log n), т.к. большинство узлов близко к листьям.

**Иллюстрации (max-heap):**

Дерево и массив (обход по уровням):
```
        9
       / \
      7   5
     / \ / \
    4  3 2  1

индекс:  0  1  2  3  4  5  6
массив: [9, 7, 5, 4, 3, 2, 1]
```

**push(6) — вставка и siftUp:** ставим 6 в конец, поднимаем вверх, пока родитель меньше.

До:
```
        9
       / \
      7   5
     / \ / \
    4  3 2  1
```

После добавления 6 в конец:
```
        9
       / \
      7   5
     / \ / \ \
    4  3 2  1  6
```

После siftUp (6 поднялась, обмен с 5; 5 ушла вниз, левый потомок 4):
```
        9
       / \
      7   6
     / \ / \
    4  3 2  1
   /
  5
```
Массив: [9,7,5,4,3,2,1] → [9,7,5,4,3,2,1,6] → [9,7,6,4,3,2,1,5].

**pop() — извлечение максимума и siftDown:** забираем корень (9), на корень ставим последний (1), просеиваем вниз, меняя с большим ребёнком.

Было:
```
        9
       / \
      7   6
     / \ / \
    4  3 2  1
```

Убрали 9, на корень поставили последний (1):
```
        1
       / \
      7   6
     / \ /
    4  3 2
```

После siftDown (1 → обмен с 7, затем с 4):
```
        7
       / \
      4   6
     / \ /
    1  3 2
```
Шаги: 1 < 7 и 1 < 6 → меняем с большим (7); 1 < 4 и 1 < 3 → меняем с 4; 1 в листе. Массив: [9,7,6,4,3,2,1] → [1,7,6,4,3,2] → [7,4,6,1,3,2].

| Операция | Оценка |
|----------|--------|
| push | O(log n) |
| pop | O(log n) |
| peek | O(1) |
| heapify | O(n) |

## 1.7. Поток данных (stream). Основные операции, способы реализации, сравнительная характеристика и асимптотика

**Что такое поток.** Поток (stream) — это абстракция **последовательного доступа** к данным: мы не прыгаем к произвольному элементу по индексу, а «идём» по данным шаг за шагом, от начала к концу. Один и тот же интерфейс (hasNext, read) можно использовать для разных источников: массив в памяти, файл на диске, данные из сети. Код, который обрабатывает поток, не знает, откуда приходят байты — это упрощает переиспользование (например, один парсер для файла и для сетевого сокета).

**Последовательный vs прямой доступ.** Прямой (random access): обращение к элементу по индексу за O(1) — массив, `arr[i]`. Последовательный: следующий элемент получаем только после предыдущего — как лента: сдвинулись вперёд, прочитали следующий байт. Поток — это как раз модель «ленты».

**Основные операции:**
- **hasNext()** (или **eof()** — end of stream) — есть ли ещё данные для чтения. Перед каждым read обычно проверяют hasNext, чтобы не читать за концом.
- **read()** — вернуть **следующий** элемент и сдвинуть «курсор» вперёд. Элемент может быть байт, символ или запись фиксированного размера — зависит от типа потока.
- Для **входного потока** только чтение. Для **выходного** — **write(x)** — записать следующий элемент.
- **Двунаправленный** поток (файл, открытый на чтение и запись) может поддерживать **seek(pos)** — перейти к позиции pos для следующего read/write. Тогда доступ не строго последовательный.

**Реализация потока из массива.** Данные уже в памяти: массив `data[]` и его `size`. Храним текущую позицию `pos` (индекс следующего элемента для чтения). **hasNext** — `pos < size`. **read()** — вернуть `data[pos]` и увеличить `pos` на 1 (т.е. `data[pos++]`). Обе операции O(1). Вся память под массив уже выделена, обращений к ОС нет. Если нужен произвольный доступ — можно добавить **seek(pos)**: просто присвоить внутренней позиции значение pos; тогда поток ведёт себя как «вид» на массив с текущей позицией.

**Реализация потока из файла.** Данные на диске; каждое обращение к файлу — системный вызов (read/write), он дорогой. **Буферизация:** вместо «один байт — один вызов» читаем **блок** (например 4 KB) в буфер в памяти. **read()** отдаёт байты из буфера по одному (или небольшими порциями); когда буфер опустел — следующий вызов read в ОС, заполняем буфер снова. Так один системный вызов даёт много байт — число обращений к диску и ядру падает. **read()** из буфера — O(1); редкие «подкачки» с диска — O(размер_блока). **seek(pos)** на файле может требовать позиционирования на диске (O(1) в кэше ОС, но при реальном сдвиге головки — дороже); для чисто последовательного чтения seek не нужен.

**Сравнение.** Массив: минимум накладных расходов, O(1) на read и seek, всё в RAM. Файл: ограничен скоростью диска и системными вызовами; буфер сглаживает это при последовательном чтении. Для последовательной обработки (парсинг, копирование) оба дают «потоковый» интерфейс; выбор зависит от того, откуда данные изначально (память vs диск/сеть) и нужен ли произвольный доступ.

## 1.8. Очередь с приоритетами. Способы реализации и оценка асимптотики

**Что такое очередь с приоритетами.** Абстрактный тип данных: элементы имеют приоритет (число или ключ). Извлекаем не «первый пришедший» (как в FIFO), а элемент с **наивысшим** (или **наименьшим**) приоритетом. Два варианта: **extractMin** — минимум приоритета первым (задачи с «меньше = срочнее»); **extractMax** — максимум первым. Остальной интерфейс тот же: вставка и просмотр экстремума без извлечения.

**Операции АТД:**
- **insert(x)** — добавить элемент x с его приоритетом.
- **extractMin()** (или **extractMax()**) — вернуть и удалить элемент с минимальным (максимальным) приоритетом. Обычно не определено для пустой очереди.
- **peek()** (или **findMin** / **findMax**) — вернуть экстремум **без** удаления. O(1), если храним его в известном месте.

**Примеры использования:** планировщик задач (срочность = приоритет), алгоритм Дейкстры (извлечение вершины с минимальным расстоянием), слияние k отсортированных последовательностей (минимум среди текущих голов — приоритет).

**Реализация на бинарной куче (min-heap для extractMin).** Куча хранит элементы по приоритету: в корне — минимум. **insert:** добавляем в конец массива (новый лист), затем **siftUp** — поднимаем, пока родитель больше. O(log n). **extractMin:** забираем корень (это минимум), на место корня ставим последний элемент массива, затем **siftDown** — опускаем, пока кто-то из детей меньше. O(log n). **peek:** корень кучи — O(1). И вставка, и извлечение за O(log n), без перебора всех элементов — удобно, когда и вставок, и извлечений много (типичный случай в алгоритмах).

**Реализация на отсортированном связном списке.** Список упорядочен по приоритету (например, от меньшего к большему). **extractMin** — удаление головы списка, O(1). **insert** — поиск места вставки (обход до первого элемента с большим приоритетом), вставка узла O(1), но поиск O(n). **peek** — голова списка, O(1). Получается: извлечение дёшево, вставка дорогая. Выгодно, если извлечений мало, а вставок много (или наоборот — тогда держим порядок «от большего к меньшему» и extractMax = голова).

**Сравнение.** Куча: insert O(log n), extractMin O(log n), peek O(1) — сбалансировано; дополнительная память — массив под кучу, без указателей на узлы. Отсортированный список: extractMin O(1), insert O(n), peek O(1) — выбор имеет смысл, когда одна из операций резко преобладает. В общем случае для очереди с приоритетами обычно используют кучу.

## 1.9. Бинарное дерево поиска. Основные операции. Балансировка. Оценка асимптотики

**Инвариант BST:** для каждого узла все ключи в левом поддереве меньше ключа узла, в правом — больше (или наоборот при другом порядке). Inorder-обход даёт отсортированную последовательность.

**Поиск:** спуск от корня — влево, если key меньше, вправо, если больше. O(h), где h — высота. В сбалансированном дереве h = O(log n), в вырожденном (цепочка) h = O(n).

**Вставка:** спуск до «провала» (nullptr), создание нового узла на этом месте. O(h).

**Удаление — три случая:** (1) нет детей — просто удалить узел; (2) один ребёнок — поднять ребёнка на место удаляемого; (3) два ребёнка — заменить ключ удаляемого на минимум правого поддерева, рекурсивно удалить тот минимум (у него нет левого сына, сводится к случаю 1 или 2). Минимум правого поддерева — единственный кандидат, сохраняющий инвариант при переносе в корень.

**Балансировка:** AVL — разница высот поддеревьев не более 1, повороты при нарушении. Красно-чёрное дерево — инварианты по цветам, перекрашивание и повороты. Гарантируют h = O(log n).

**Иллюстрации:**

Инвариант BST: слева от узла — меньшие, справа — большие. Inorder (левый → узел → правый) даёт порядок по возрастанию.
```
          5
         / \
        3   8
       / \   \
      1   4   9

Inorder: 1, 3, 4, 5, 8, 9
```

**Поиск (например, 4):** спуск от корня — 4 < 5 → влево; 4 > 3 → вправо; нашли 4.
```
          5
         / \     ищем 4: 5 → 3 → 4
        3   8
       / \   \
      1   4   9
```

**Вставка (6):** спуск до «провала» — 6 > 5 → вправо; 6 < 8 → влево; у 8 нет левого сына — вешаем 6.
```
  До:        5      После вставки 6:    5
           / \                        / \
          3   8                      3   8
         / \   \                    / \ / \
        1   4   9                  1  4 6  9
```

**Удаление — три случая:**

(1) Узел без детей — просто убрать ссылку.
```
  Удаляем 4:    5            Стало:    5
               / \                    / \
              3   8                  3   8
             / \   \                /   \
            1   4   9              1    9
```

(2) Один ребёнок — поднять ребёнка на место удаляемого.
```
  Удаляем 8:    5            Стало:    5
               / \                    / \
              3   8                  3   9
             / \   \                / \
            1   4   9              1   4
```

(3) Два ребёнка — заменить ключ удаляемого на минимум правого поддерева, удалить тот минимум (у него нет левого сына).
```
  Удаляем 5:    5            Минимум правого (поддерево 8): 6.
               / \           5 заменяем на 6, удаляем старую 6:
              3   8
             / \ / \
            1  4 6  9

  Стало:        6
             /   \
            3     8
           / \     \
          1   4     9
```

**Вырожденное дерево (цепочка)** — высота O(n), поиск/вставка O(n). **Сбалансированное** — высота O(log n), операции O(log n).
```
Вырожденное:    1     Сбалансированное:    5
                 \                        / \
                  2                      3   8
                   \                    / \   \
                    3                  1   4   9
                     \
                      4
```

## 1.10. Понятие итератора, основные операции

**Что такое итератор.** Итератор — объект, который обозначает **позицию** в контейнере (или в последовательности), а не сам элемент. Через итератор можно получить элемент в этой позиции и перейти к следующей (или предыдущей). Так один и тот же код обхода (цикл с `++it`, `*it`, `it != end`) работает для массива, списка, множества — контейнер скрыт за единым интерфейсом. Алгоритмы стандартной библиотеки (find, sort, copy) принимают пару итераторов «начало — конец» и не зависят от типа контейнера.

**Итератор и конец последовательности.** У контейнера есть **begin()** — итератор на первый элемент, и **end()** — итератор «за последним» (past-the-end). Элемента в позиции end() нет; эта позиция нужна для проверки «дошли до конца»: цикл `for (it = begin(); it != end(); ++it)` обходит все элементы. Разыменовывать end() нельзя.

**Основные операции (минимум для обхода вперёд):**
- **Разыменование** `*it` — доступ к элементу в текущей позиции. Для изменяемого контейнера `*it = x` записывает в эту позицию.
- **Инкремент** `++it` (или `it++`) — сдвиг итератора к следующему элементу.
- **Сравнение** `it != end` (или `it == end`) — проверка, достигли ли конца. Две итераторы одного контейнера можно сравнивать на равенство.

Этого достаточно, чтобы пройти по последовательности один раз (категория Input или Forward).

**Дополнительные операции по категориям:**
- **Bidirectional** — движение назад: `--it` (или `it--`). Пример: двусвязный список, std::list.
- **Random Access** — произвольный сдвиг и доступ по «индексу»: `it + k`, `it - k`, `it += k`, `it[k]` (эквивалент `*(it + k)`), сравнение порядка `it1 < it2`. Сложность — O(1). Пример: массив, std::vector, std::deque.

**Категории итераторов (от слабой к сильной):**
- **Input** — один проход вперёд, только чтение. После `++it` старые копии итератора могут стать невалидными. Пример: чтение из потока (stream iterator).
- **Output** — один проход вперёд, только запись. Аналогично, один проход.
- **Forward** — многократный проход вперёд, чтение (и при необходимости запись). Можно сохранить итератор и пройти диапазон несколько раз. Пример: односвязный список.
- **Bidirectional** — всё то же + движение назад (`--it`). Пример: std::list.
- **Random Access** — всё то же + сдвиг на k за O(1), `it[k]`, сравнение порядка. Пример: std::vector, std::deque.

**Зачем категории.** Алгоритм указывает **минимальную** категорию итераторов, с которой он работает. Тогда один и тот же алгоритм можно вызывать для любого контейнера, предоставляющего не меньшую категорию. Примеры: **std::sort** требует Random Access (нужен произвольный доступ за O(1)); **std::reverse** — Bidirectional (достаточно движения вперёд и назад); **std::find** — Input (достаточно одного прохода вперёд и чтения). Для std::list нельзя вызвать sort по умолчанию (у list итераторы Bidirectional, не Random Access); для vector — можно.

**Пример кода (C++):**
```cpp
#include <vector>
#include <list>
#include <algorithm>

// Обход через итераторы — один и тот же стиль для vector и list
std::vector<int> v = {3, 1, 4, 1, 5};
for (auto it = v.begin(); it != v.end(); ++it) {
    // *it — текущий элемент
    *it += 1;  // запись через итератор
}

// Алгоритм принимает пару итераторов [begin, end)
auto found = std::find(v.begin(), v.end(), 6);  // ищем 6
if (found != v.end())
    *found = 0;  // нашли — разыменовали и изменили

// Random Access: сдвиг и it[k] — только у vector, deque
auto mid = v.begin() + v.size() / 2;  // итератор на середину
int x = v.begin()[2];                 // третий элемент, как *(v.begin() + 2)

std::sort(v.begin(), v.end());        // OK: vector даёт Random Access

std::list<int> lst = {3, 1, 4};
// std::sort(lst.begin(), lst.end()); // ошибка: list — Bidirectional, не Random Access
for (auto it = lst.begin(); it != lst.end(); ++it) { /* обход вперёд */ }
auto rit = lst.end();
--rit;  // Bidirectional: движение назад
```
В цикле `auto it = v.begin()` — тип итератора (например, `std::vector<int>::iterator`) выводится компилятором; `begin()`/`end()` задают полуинтервал [begin, end).

## 1.11. Тип Option&lt;&gt;

**Что такое Option.** Option (или Maybe) — тип, представляющий **значение, которое может отсутствовать**. Вместо «указатель может быть nullptr» или «бросить исключение при отсутствии» мы явно храним один из двух вариантов: **Some(value)** — значение есть, или **None** — значения нет. Тип в сигнатуре функции тогда ясно говорит: «результат может отсутствовать» (например, поиск в контейнере), и компилятор заставляет обработать оба случая — нельзя случайно разыменовать «пустой» результат.

**Зачем не nullptr.** Указатель T* тоже может быть «пустым» (nullptr), но тип не различает «указатель на один объект» и «указатель на массив»; при этом nullptr применим только к указателям. Option&lt;T&gt; подходит для любого типа T (в том числе для значений: int, string, объект по значению). Кроме того, Option явно выражает намерение: «здесь может не быть результата», а не «здесь адрес».

**Основные операции:**
- **Проверка наличия** — has_value() (или is_some()) возвращает true, если значение есть; иначе false.
- **Получить значение** — value() или *opt; при вызове для None — неопределённое поведение (или исключение в std::optional). Безопаснее сначала проверить has_value() или использовать value_or(default).
- **Значение по умолчанию** — value_or(x): если значение есть, вернуть его; иначе x. Удобно, когда при отсутствии результата нужен запасной вариант.
- **Условное извлечение** — if (opt) { use(*opt); } или pattern matching в языках с поддержкой (Rust, Haskell).

**Реализация (идея).** Храним union: либо значение типа T, либо «пустое» состояние; плюс флаг has_value. Для не-POD типов T при копировании/присваивании optional нужно вызывать конструктор/деструктор в месте значения — через placement new и явный вызов destructor.

**Реализация Optional (код, аналог std::optional):**
```cpp
#include <new>
#include <stdexcept>
#include <utility>

struct nullopt_t { explicit nullopt_t() = default; };
constexpr nullopt_t nullopt{};

template<typename T>
class Optional {
public:
    Optional() : has_value_(false) {}
    Optional(nullopt_t) : has_value_(false) {}
    Optional(const T& value) : has_value_(true) { new (storage_) T(value); }
    Optional(T&& value) : has_value_(true) { new (storage_) T(std::move(value)); }
    Optional(const Optional& o) : has_value_(o.has_value_) {
        if (o.has_value_) new (storage_) T(*o.ptr());
    }
    Optional(Optional&& o) noexcept : has_value_(o.has_value_) {
        if (o.has_value_) { new (storage_) T(std::move(*o.ptr())); o.destroy(); }
    }
    ~Optional() { destroy(); }

    Optional& operator=(nullopt_t) { destroy(); return *this; }
    Optional& operator=(const Optional& o) {
        if (this != &o) { destroy(); if (o.has_value_) { new (storage_) T(*o.ptr()); has_value_ = true; } }
        return *this;
    }
    Optional& operator=(Optional&& o) noexcept {
        if (this != &o) { destroy(); if (o.has_value_) { new (storage_) T(std::move(*o.ptr())); has_value_ = true; o.destroy(); } }
        return *this;
    }
    Optional& operator=(const T& value) { destroy(); new (storage_) T(value); has_value_ = true; return *this; }
    Optional& operator=(T&& value) { destroy(); new (storage_) T(std::move(value)); has_value_ = true; return *this; }

    bool has_value() const { return has_value_; }
    explicit operator bool() const { return has_value_; }
    T& value() { if (!has_value_) throw std::bad_optional_access(); return *ptr(); }
    const T& value() const { if (!has_value_) throw std::bad_optional_access(); return *ptr(); }
    T& operator*() { return *ptr(); }
    const T& operator*() const { return *ptr(); }
    T* operator->() { return ptr(); }
    const T* operator->() const { return ptr(); }
    T value_or(T default_val) const { return has_value_ ? *ptr() : default_val; }
    void reset() { destroy(); }
    template<typename... Args> T& emplace(Args&&... args) { destroy(); new (storage_) T(std::forward<Args>(args)...); has_value_ = true; return *ptr(); }
    void swap(Optional& o) {
        if (has_value_ && o.has_value_) { std::swap(*ptr(), *o.ptr()); return; }
        if (has_value_) { o.emplace(std::move(*ptr())); destroy(); return; }
        if (o.has_value_) { emplace(std::move(*o.ptr())); o.destroy(); }
    }

private:
    alignas(T) unsigned char storage_[sizeof(T)];
    bool has_value_;
    T* ptr() { return reinterpret_cast<T*>(storage_); }
    const T* ptr() const { return reinterpret_cast<const T*>(storage_); }
    void destroy() { if (has_value_) { ptr()->~T(); has_value_ = false; } }
};

template<typename T> Optional<T> make_optional(T value) { return Optional<T>(std::move(value)); }
template<typename T> void swap(Optional<T>& a, Optional<T>& b) { a.swap(b); }
// Сравнения и hash — по аналогии с std::optional (сравнить has_value_, затем *ptr()).
```
В стандартной библиотеке используется похожая схема (буфер под T + флаг, placement new, явный вызов деструктора). Исключение при пустом value() — std::bad_optional_access.

**Реализация в стандартной библиотеке: std::optional (C++17) — полный API.**
```cpp
#include <optional>

// --- Константа ---
std::nullopt;  // пустое состояние (тип std::nullopt_t)

// --- Конструкторы ---
std::optional<int> o1;                    // по умолчанию: пусто
std::optional<int> o2 = std::nullopt;      // пусто
std::optional<int> o3 = 42;               // из значения (implicit)
std::optional<int> o4{42};                // из значения
std::optional<int> o5 = o3;               // копирование
std::optional<int> o6 = std::move(o4);     // перемещение

// --- Функции создания ---
auto o7 = std::make_optional(42);         // make_optional(args...)
auto o8 = std::make_optional<int>();      // optional с value-initialized T

// --- Методы ---
o3.has_value();       // bool: есть ли значение
bool(o3);             // то же (operator bool)
o3.value();           // T&: значение (при пустом — std::bad_optional_access)
*o3;                  // T&: разыменование (при пустом — UB)
o3.value_or(0);       // T: значение или default
o3.reset();           // сделать пустым (уничтожить значение)
o3.emplace(7);        // уничтожить старое, создать T(7) in-place
o3.swap(o5);          // обмен содержимым

// --- Присваивание ---
o1 = std::nullopt;    // очистить
o1 = 10;              // присвоить значение
o1 = o2;              // копировать optional
o1 = std::move(o2);   // переместить optional

// --- Свободные функции ---
std::swap(o1, o2);   // обмен
o1 == o2; o1 != o2; o1 < o2; o1 <= o2; o1 > o2; o1 >= o2;  // сравнение
o1 == std::nullopt; o1 != std::nullopt;
std::hash<std::optional<int>>{}(o1);  // хеш (пустой и непустой различаются)
```
**Пример кода (std::optional):**
```cpp
#include <optional>
#include <iostream>
#include <vector>

// Функция возвращает optional — явно «может не найти»
std::optional<int> find_first_even(const std::vector<int>& v) {
    for (int x : v)
        if (x % 2 == 0)
            return x;   // return std::optional{x};
    return std::nullopt; // или return {};
}

std::vector<int> vec = {1, 3, 5, 6, 7};
auto result = find_first_even(vec);

if (result.has_value())
    printf("первое чётное: %d\n", *result);
else
    printf("чётных нет\n");

// Значение по умолчанию
int value = result.value_or(-1);

// Создание и проверка
std::optional<std::string> name = "Alice";
std::optional<std::string> empty = std::nullopt;
if (name)
    std::cout << *name << '\n';
```
Тип возвращаемого значения find_first_even — **std::optional&lt;int&gt;**; вызывающий код обязан учитывать случай «ничего не найдено», иначе разыменование пустого optional — ошибка. В стандартной библиотеке optional также возвращают, например, **std::map::find** не возвращает optional (возвращает итератор), но **std::optional** — стандартный способ выразить «результат может отсутствовать» в C++17.

## 1.12. Тип Either&lt;&gt;

**Что такое Either.** Either (или Sum type) — тип, представляющий **одно из двух значений**: либо значение типа L (**Left(l)**), либо значение типа R (**Right(r)**). В любой момент хранится ровно один из двух вариантов. В отличие от Option, «отсутствие» не просто «ничего», а **альтернативное значение** — например, сообщение об ошибке или код причины. Так функция может вернуть «либо результат T, либо описание ошибки E», и вызывающий код обязан обработать оба случая.

**Either и Option.** Option&lt;T&gt; — «значение T или ничего». Either&lt;L, R&gt; — «значение L или значение R». Option можно выразить через Either: «значение или пустой маркер» (Either&lt;Unit, T&gt;). Either удобен, когда при «неуспехе» нужно передать причину: ошибку, код, строку — и обработать её без исключений.

**Соглашение Result (Left = ошибка, Right = успех).** Часто используют как **Result&lt;T, E&gt;** (или Either&lt;E, T&gt;): **Left** — ошибка типа E (сообщение, код, структура с полями), **Right** — успешный результат типа T. По соглашению «Right = правильный результат» (right = правый, правильный). Тогда возврат ошибки из функции — не исключение, а значение Left(err); вызывающий проверяет «есть ли ошибка» и обрабатывает её явно (логирование, повтор, запасной путь). Исключения остаются для действительно исключительных ситуаций.

**Основные операции:**
- **Проверка варианта** — is_left() / is_right() (или index() — какой вариант активен). По ним решаем, разыменовывать левое или правое значение.
- **Доступ к значению** — left() и right() (или get&lt;0&gt;, get&lt;1&gt; в variant); при неверном варианте — неопределённое поведение или исключение. Безопаснее сначала проверить вариант или использовать visit/match.
- **Обработка обоих случаев** — match (pattern matching): «если Left — сделать с err одно, если Right — сделать с value другое». В C++ это можно реализовать через std::visit с variant.

**Реализация (идея).** Храним union из двух полей (L и R) и дискриминатор (какой вариант активен). При копировании/присваивании нужно корректно вызывать конструкторы и деструкторы для активного варианта.

**Реализация Variant&lt;L, R&gt; (код, аналог std::variant для двух типов):**
```cpp
#include <new>
#include <stdexcept>
#include <utility>
#include <cstddef>

template<typename L, typename R>
class Variant {
public:
    Variant() : index_(0) { new (storage_) L(); }  // по умолчанию — первый тип
    Variant(const L& l) : index_(0) { new (storage_) L(l); }
    Variant(L&& l) : index_(0) { new (storage_) L(std::move(l)); }
    Variant(const R& r) : index_(1) { new (storage_) R(r); }
    Variant(R&& r) : index_(1) { new (storage_) R(std::move(r)); }
    Variant(const Variant& o) : index_(o.index_) {
        if (index_ == 0) new (storage_) L(*o.ptr_l());
        else new (storage_) R(*o.ptr_r());
    }
    Variant(Variant&& o) noexcept : index_(o.index_) {
        if (index_ == 0) { new (storage_) L(std::move(*o.ptr_l())); o.set_default(); }
        else { new (storage_) R(std::move(*o.ptr_r())); o.set_default(); }
    }
    ~Variant() { destroy(); }

    Variant& operator=(const Variant& o) {
        if (this != &o) { destroy(); index_ = o.index_; if (index_ == 0) new (storage_) L(*o.ptr_l()); else new (storage_) R(*o.ptr_r()); }
        return *this;
    }
    Variant& operator=(Variant&& o) noexcept {
        if (this != &o) { destroy(); index_ = o.index_; if (index_ == 0) { new (storage_) L(std::move(*o.ptr_l())); } else { new (storage_) R(std::move(*o.ptr_r())); } o.set_default(); }
        return *this;
    }

    size_t index() const { return index_; }
    bool valueless_by_exception() const { return false; }  // упрощённо

    L* ptr_l() { return reinterpret_cast<L*>(storage_); }
    const L* ptr_l() const { return reinterpret_cast<const L*>(storage_); }
    R* ptr_r() { return reinterpret_cast<R*>(storage_); }
    const R* ptr_r() const { return reinterpret_cast<const R*>(storage_); }

private:
    static constexpr size_t size_ = sizeof(L) > sizeof(R) ? sizeof(L) : sizeof(R);
    static constexpr size_t align_ = alignof(L) > alignof(R) ? alignof(L) : alignof(R);
    alignas(align_) unsigned char storage_[size_];
    size_t index_;
    void destroy() {
        if (index_ == 0) ptr_l()->~L();
        else ptr_r()->~R();
    }
    void set_default() { destroy(); index_ = 0; new (storage_) L(); }  // после move оставить в валидном состоянии
};

template<typename T, typename L, typename R> bool holds_alternative(const Variant<L,R>& v) {
    if constexpr (std::is_same_v<T, L>) return v.index() == 0;
    else return v.index() == 1;
}
template<typename T, typename L, typename R> T& get(Variant<L,R>& v) {
    if constexpr (std::is_same_v<T, L>) { if (v.index() != 0) throw std::bad_variant_access(); return *v.ptr_l(); }
    else { if (v.index() != 1) throw std::bad_variant_access(); return *v.ptr_r(); }
}
template<typename T, typename L, typename R> T* get_if(Variant<L,R>* v) {
    if (!v) return nullptr;
    if constexpr (std::is_same_v<T, L>) return v->index() == 0 ? v->ptr_l() : nullptr;
    else return v->index() == 1 ? v->ptr_r() : nullptr;
}
template<typename Visitor, typename L, typename R>
decltype(auto) visit(Visitor&& vis, Variant<L,R>& v) {
    if (v.index() == 0) return std::forward<Visitor>(vis)(*v.ptr_l());
    else return std::forward<Visitor>(vis)(*v.ptr_r());
}
// std::bad_variant_access — в <variant>; здесь предполагается определённым или заменён на std::runtime_error.
```
В стандартной библиотеке std::variant поддерживает произвольное число типов (рекурсия или массив указателей на destroy/copy/move), in_place_index/in_place_type и учёт исключений (valueless_by_exception).

**Реализация Expected&lt;T, E&gt; (код, аналог std::expected, C++23):**
```cpp
template<typename T, typename E>
class Expected {
public:
    Expected(const T& t) : has_value_(true) { new (storage_) T(t); }
    Expected(T&& t) : has_value_(true) { new (storage_) T(std::move(t)); }
    Expected(const E& e) : has_value_(false) { new (err_) E(e); }
    Expected(E&& e) : has_value_(false) { new (err_) E(std::move(e)); }
    ~Expected() { if (has_value_) ptr()->~T(); else ptr_err()->~E(); }
    bool has_value() const { return has_value_; }
    explicit operator bool() const { return has_value_; }
    T& value() { if (!has_value_) throw std::bad_expected_access<E>(*ptr_err()); return *ptr(); }
    E& error() { return *ptr_err(); }
    T value_or(T default_val) const { return has_value_ ? *ptr() : default_val; }
private:
    alignas(T) unsigned char storage_[sizeof(T)];
    alignas(E) unsigned char err_[sizeof(E)];
    bool has_value_;
    T* ptr() { return reinterpret_cast<T*>(storage_); }
    E* ptr_err() { return reinterpret_cast<E*>(err_); }
};
// std::bad_expected_access<E> — в <expected>. Копирование/перемещение/swap/and_then/transform — по аналогии с Optional и variant.
```

**Реализация в стандартной библиотеке — полный API.**

**std::variant&lt;Types...&gt; (C++17), заголовок `<variant>`:**
```cpp
#include <variant>
#include <string>

using V = std::variant<std::string, int>;

// --- Конструкторы ---
V v1;                          // по умолчанию: первый тип value-initialized
V v2 = 42;                     // из значения (индекс выбирается по типу)
V v3 = std::string("err");     // из значения
V v4 = v2;                     // копирование
V v5 = std::move(v3);          // перемещение
V v6{std::in_place_index<0>, "hello"};   // in-place по индексу
V v7{std::in_place_type<int>, 7};       // in-place по типу

// --- Методы ---
v2.index();                    // size_t: индекс активного типа (0 или 1)
v2.valueless_by_exception();    // bool: в невалидном состоянии после исключения

// --- Свободные функции ---
std::holds_alternative<int>(v2);        // true — активен int
std::holds_alternative<std::string>(v2); // false
std::get<int>(v2);              // int&: доступ по типу (иначе исключение)
std::get<1>(v2);                // int&: доступ по индексу
std::get_if<int>(&v2);          // int* или nullptr
std::get_if<std::string>(&v2);  // nullptr
std::visit([](auto&& x) { /* обработать любой вариант */ }, v2);
std::swap(v2, v3);
v2 == v3; v2 != v3; v2 < v3;    // сравнение (по индексу, затем по значению)
std::hash<V>{}(v2);             // хеш (если все типы hashable)
```

**std::expected&lt;T, E&gt; (C++23), заголовок `<expected>`:**
```cpp
#include <expected>
#include <string>

std::expected<int, std::string> e1 = 42;              // значение
std::expected<int, std::string> e2 = std::unexpected("err"); // ошибка
auto e3 = std::expected<int, std::string>(std::unexpected("fail"));

// --- Методы ---
e1.has_value();     // bool
bool(e1);           // operator bool
e1.value();         // T& (при ошибке — исключение)
e1.error();         // E& (при значении — не вызывать)
e1.value_or(0);     // T: значение или default
e1.value_or_else([](const std::string&) { return 0; }); // T или f(error)
e1.and_then([](int n) { return std::expected<int, std::string>(n+1); }); // цепочка
e1.or_else([](std::string) { return std::expected<int, std::string>(0); });
e1.transform([](int n) { return n*2; });  // expected<U,E>
e1.transform_error([](std::string s) { return s + "!"; });
e1.swap(e2);
e1.emplace(7);     // заменить на значение 7
e1.reset();        // заменить на ошибку (value-initialized E)

// --- Свободные функции ---
std::swap(e1, e2);
e1 == e2; e1 != e2;  // сравнение
std::hash<std::expected<int, std::string>>{}(e1);
```
**Пример кода (std::variant как Either; C++23 — std::expected):**
```cpp
#include <variant>
#include <string>
#include <cstdlib>

// Result<int, std::string>: успех — int, ошибка — строка
using ParseResult = std::variant<std::string, int>;

// «Парсинг» строки в int: либо число, либо сообщение об ошибке
ParseResult parse_int(const std::string& s) {
    if (s.empty())
        return std::string("пустая строка");
    char* end = nullptr;
    long v = std::strtol(s.c_str(), &end, 10);
    if (end != s.c_str() + s.size())
        return std::string("не число");
    if (v < -1000 || v > 1000)
        return std::string("вне диапазона");
    return static_cast<int>(v);
}

ParseResult r = parse_int("42");

if (std::holds_alternative<int>(r))
    printf("число: %d\n", std::get<int>(r));
else
    printf("ошибка: %s\n", std::get<std::string>(r).c_str());

// По соглашению: первый тип в variant — «ошибка», второй — «результат»
// Тогда Left = get<0>, Right = get<1>
```
В C++23 можно использовать **std::expected&lt;int, std::string&gt;** — там явно value() и error(), без variant по индексу. Идея та же: функция возвращает «либо значение, либо ошибку», вызывающий обязан проверить и обработать оба случая без исключений.

## 1.13. N-арные деревья. B- и B+-деревья. Основные операции, оценка асимптотики

**Обозначение.** Это **три разные вещи**: **B дерево** (англ. B-tree) — общее семейство сбалансированных многопутевых деревьев поиска; **B минус дерево** (B−) — одна вариация B дерева; **B плюс дерево** (B+) — вторая вариация. B минус и B плюс — два варианта внутри семейства B дерева.

**N-арное дерево (общее определение).** N-арное дерево — дерево, у которого у каждого узла **не более N детей** (N ≥ 2). Узел хранит данные (например, один ключ или значение) и до N указателей на дочерние узлы. При N = 2 получается двоичное дерево (левый и правый ребёнок). При N > 2 узел может иметь 1, 2, …, N детей — дерево «шире», при том же числе узлов высота меньше, чем у двоичного. Примеры: файловая система (каталог — узел, подкаталоги — дети), дерево игры (ход — узел, варианты ответа — дети). N-арное дерево само по себе **не задаёт порядка** на ключах; порядок и инварианты поиска задаются конкретной структурой (например, B дерево и его вариации).

**B дерево (общее).** B дерево — **сбалансированное дерево поиска**, в котором один узел может хранить **много ключей** и иметь **много детей**. Структура оптимизирована под хранение на диске: узел соответствует одной странице (блоку); за одно обращение к диску читается целый узел с несколькими ключами. Ниже описаны две вариации: **B минус дерево** и **B плюс дерево**.

**B минус дерево (B−) порядка t (t ≥ 2).** Одна из вариаций B дерева. Ключи (и при необходимости данные) могут находиться **в любом узле** — и во внутренних, и в листьях.

- **Инварианты узла:** в каждом **внутреннем** узле (кроме корня) от **t** до **2t** ключей и от **t+1** до **2t+1** указателей на детей. В **корне** — от 1 до 2t ключей и от 2 до 2t+1 детей (если корень не лист). Ключи в узле упорядочены: k₀ < k₁ < … < k_{m-1}. Поддерево с индексом 0 содержит ключи **меньше** k₀; поддерево с индексом i (1 ≤ i < m) — ключи **между** k_{i−1} и k_i; поддерево с индексом m — ключи **больше** k_{m−1}.
- **Листья:** все листья на **одной глубине** — дерево сбалансировано по высоте.
- **Высота:** O(log_t n), где n — число ключей. При большом t в узле много ключей, высота мала — мало обращений к диску.
- **Поиск:** спуск от корня. В текущем узле ищем ключ среди k₀,…,k_{m−1} (бинарный или линейный поиск). Если нашли — возвращаем узел и индекс. Если не нашли и узел — лист — ключа нет. Иначе переходим в соответствующее поддерево (то, в диапазон которого попадает искомый ключ). Время O(log_t n) = O(log n) при фиксированном t.
- **Вставка:** ищем лист, в который должен попасть ключ; вставляем ключ в этот узел. Если в узле стало больше 2t ключей — **разбиение (split):** средний ключ переносим в родителя, узел разбивается на два узла (по t ключей в каждом). Если переполнился корень — создаётся новый корень с одним ключом, глубина дерева увеличивается на 1. Время O(log_t n).
- **Удаление:** аналогично — поиск ключа, удаление из узла; при недозаполнении узла (меньше t ключей у не-корня) — слияние с соседом или перераспределение ключей. O(log_t n).

**B плюс дерево (B+).** Вторая вариация B дерева, часто используемая в СУБД и файловых системах.

- **Данные только в листьях:** во **внутренних** узлах хранятся только **копии ключей** (разделители) для маршрутизации поиска; фактические пары (ключ, значение) или указатели на записи хранятся **только в листьях**.
- **Листья связаны в двусвязный список** (в порядке возрастания ключей). Последовательный обход по ключам — обход списка листьев без спуска по дереву. Диапазонные запросы «все записи с ключом от a до b»: поиск листа с ключом a, затем движение по списку листьев до ключа b. Поиск начала O(log n), обход k элементов O(k).
- **Поиск:** O(log n). **Вставка:** поиск нужного листа, вставка в лист; при переполнении — split листа, копия разделяющего ключа поднимается в родителя. O(log n).

**Пример кода: структура узла B минус дерева и поиск.**
```cpp
#include <vector>

const int t = 2;  // порядок B минус дерева: в не-корне от t до 2t ключей

struct BTreeNode {
    std::vector<int> keys;                      // ключи в узле (упорядочены)
    std::vector<BTreeNode*> children;          // указатели на детей (у листа пусто)
    bool leaf;

    BTreeNode(bool is_leaf) : leaf(is_leaf) {}
};

// Поиск key в поддереве с корнем node.
// Возвращает (узел, индекс ключа) при успехе, (nullptr, -1) при отсутствии.
std::pair<BTreeNode*, int> search(BTreeNode* node, int key) {
    size_t i = 0;
    while (i < node->keys.size() && key > node->keys[i])
        ++i;
    if (i < node->keys.size() && key == node->keys[i])
        return {node, static_cast<int>(i)};
    if (node->leaf)
        return {nullptr, -1};
    return search(node->children[i], key);  // ключ в i-м поддереве: между k_{i-1} и k_i
}
```
Полная реализация B минус дерева включает вставку с разбиением узла, удаление со слиянием узлов и при необходимости работу с диском (буферизация страниц).

## 1.14. Хеш-таблицы. Хеш-функции. Разрешение коллизий в хеш-таблицах

**Принцип.** Хеш-таблица — массив **бакетов** (ячеек). Ключ преобразуется **хеш-функцией** h в индекс: `index = h(key) % m` (или другая нормализация по размеру таблицы m). В идеале каждый ключ попадает в свой бакет — доступ по ключу за O(1). **Коллизия** — разные ключи дают один и тот же индекс; их нужно как-то различать внутри бакета или искать другую свободную ячейку.

**Требования к хеш-функции:** (1) **Детерминированность** — один и тот же ключ всегда даёт один и тот же хеш. (2) **Быстрота** — вычисление за O(длина ключа) или лучше. (3) **Равномерность** — значения хеша по возможности равномерно распределены по диапазону, чтобы коллизий было мало. Часто используют модуль по простому числу или маске для устранения систематических смещений.

**Примеры хеш-функций для строк:** **djb2** — `h = 33 * h + (unsigned char)c` по символам; **FNV-1a** — xor с текущим хешем, затем умножение на простое (FNV_prime). Ниже описан **полиномиальный хеш** — удобен для строк и подстрок.

---

**Параметры хеш-таблицы.**

- **m (capacity)** — число бакетов (размер массива). Обычно берут **простое** или степень двойки (тогда индекс = `hash & (m-1)` при m = 2^k). Слишком маленькое m — много коллизий; слишком большое — лишняя память.
- **n** — текущее число элементов в таблице.
- **Load factor (коэффициент заполнения)** α = n / m — среднее число элементов на бакет. При **α > 0.7–1.0** (зависит от схемы) растёт число коллизий и длина цепочек/проб; производительность падает. Поддерживают α в заданных пределах (например, α_max ≈ 0.75).
- **Rehash** — при достижении порога (например α > 0.75) создают новую таблицу с большим m (часто m_new = 2*m или ближайшее простое), пересчитывают индекс для каждого элемента и вставляют в новую таблицу. Стоимость O(n); делают редко, амортизированно вставка остаётся O(1).
- **Начальная ёмкость** — чтобы не делать rehash слишком часто при первых вставках, таблицу часто инициализируют с минимальным m (например, 16 или 32).

---

**Способы разрешения коллизий.**

**1. Метод цепочек (chaining, закрытая адресация).** Каждый бакет — структура, хранящая все элементы с данным индексом: **список** пар (ключ, значение) или маленькое дерево. При коллизии новый элемент **добавляют в список** этого бакета. Поиск: вычислить индекс, обойти список бакета, сравнить ключи. Удаление: найти узел в списке и удалить. Плюсы: реализация проста, таблица не «переполняется» — можно хранить больше n элементов при том же m (но α растёт и поиск замедляется). Минусы: дополнительные указатели, хуже кэш. В среднем при равномерном хеше длина цепочки ≈ α; поиск (успешный и неуспешный) O(1 + α). При α = O(1) получаем O(1) в среднем.

**2. Открытая адресация (open addressing).** Все элементы хранятся **в одном массиве**; каждый слот либо пуст, либо занят парой (ключ, значение). При коллизии (слот занят другим ключом) ищут следующий свободный слот по правилу **пробирования (probing)**. Поиск: вычислить индекс, смотреть слот; если ключ не совпал — идти по правилу пробирования, пока не найдём ключ или пустой слот. Вставка: найти слот с таким ключом или первый пустой, вставить. Удаление: если просто очистить слот, поиск может «оборваться» раньше (элементы были вставлены после удалённого по цепочке проб). Поэтому удалённые слоты помечают **tombstone** (маркер «здесь было удаление»): при поиске tombstone считают занятым (идём дальше), при вставке в tombstone можно положить новый элемент. Тогда поиск остаётся корректным.

**Виды пробирования:**
- **Линейное:** следующая ячейка `(h(k) + i) % m`, i = 0, 1, 2, … Просто, но **первичная кластеризация** — подряд занятые ячейки слипаются, средняя длина проб растёт.
- **Квадратичное:** `(h(k) + c1*i + c2*i²) % m` (часто c1=0, c2=1: `(h(k) + i²) % m`). Кластеризация слабее, но **вторичная кластеризация** — ключи с одинаковым h(k) идут по одной и той же последовательности. Нужно выбирать m и константы так, чтобы пробой обходились все слоты (например, m — степень двойки для (h + i²) mod m не всегда покрывает все индексы).
- **Двойное хеширование:** следующая ячейка `(h1(k) + i * h2(k)) % m`. Вторая хеш-функция h2(k) не должна давать 0; часто берут `h2(k) = 1 + (k % (m-1))`, чтобы шаг был от 1 до m-1. Даёт хорошее распределение, мало кластеризации. При открытой адресации α не может превышать 1 (всего m слотов); обычно держат α < 0.7–0.8.

---

**Полиномиальный хеш (для строк и подстрок).** Строку s длины n рассматривают как последовательность кодов символов (например, s[0], s[1], …, s[n-1]). Выбирают **основание** p (часто простое: 31, 37, 257) и **модуль** M (большое простое или 2^64 при беззнаковой арифметике). Полиномиальный хеш:

**h(s) = (s[0]·p^0 + s[1]·p^1 + … + s[n-1]·p^(n-1)) mod M**

или (эквивалентно по коллизиям при другом порядке) **h(s) = (s[0] + s[1]·p + s[2]·p² + … + s[n-1]·p^(n-1)) mod M.**

Считают за O(n) один раз для всей строки; можно предподсчитать степени p. Удобно тем, что **хеш подстроки** s[l..r) можно выразить через хеши префиксов: если есть h(prefix(0..r)) и h(prefix(0..l)), то хеш s[l..r) = (h_r − h_l·p^(r−l)) mod M при соответствующем определении префиксных хешей. Тогда сравнение подстрок за O(1) после предподсчёта (при равенстве хешей проверяют на коллизию по необходимости). **Rolling hash:** при сдвиге окна на один символ (добавили справа, убрали слева) новый хеш пересчитывается за O(1) из старого (умножение на p, прибавление нового символа, вычитание старого с весом p^(len-1)). Используется в алгоритмах Рабина–Карпа, в хеш-таблицах для строковых ключей.

**Пример кода (полиномиальный хеш для строки, модуль 2^64):**
```cpp
#include <string>
#include <vector>

using U64 = unsigned long long;
const U64 P = 31;
const U64 M = (1ULL << 61) - 1;  // большой простой или (1ULL << 64) при переполнении по модулю 2^64

// Полиномиальный хеш: h = s[0] + s[1]*P + s[2]*P^2 + ... (mod M)
U64 poly_hash(const std::string& s) {
    U64 h = 0;
    U64 p_pow = 1;
    for (unsigned char c : s) {
        h = (h + c * p_pow) % M;
        p_pow = (p_pow * P) % M;
    }
    return h;
}

// С предподсчётом степеней P (для быстрого хеша подстрок)
std::vector<U64> prefix_hash(const std::string& s) {
    std::vector<U64> h(s.size() + 1), p_pow(s.size() + 1);
    h[0] = 0;
    p_pow[0] = 1;
    for (size_t i = 0; i < s.size(); ++i) {
        h[i + 1] = (h[i] + (unsigned char)s[i] * p_pow[i]) % M;
        p_pow[i + 1] = (p_pow[i] * P) % M;
    }
    return h;  // p_pow тоже сохраняют для хеша подстроки [l, r)
}
// Хеш подстроки s[l..r): (h[r] - h[l] * p_pow[r-l]) % M (с учётом знака при mod)
```
На практике часто берут M = 2^64 и не делают явный mod — переполнение unsigned даёт модуль 2^64 (при p нечётном распределение приемлемое).

## 1.15. Общие алгоритмы на перечислимых контейнерах: map, reduce, where

**Что такое map, reduce, where.** Это базовые операции над **перечислимыми** последовательностями (список, массив, диапазон итераторов): один проход по элементам, без произвольного доступа по индексу. Они универсальны — один и тот же интерфейс для разных контейнеров; в стандартной библиотеке C++ им соответствуют std::transform, свёртки (std::accumulate, reduce в C++17) и std::copy_if.

**map(f, xs)** — применить функцию **f** к каждому элементу последовательности **xs**, вернуть новую последовательность той же длины. Порядок сохраняется: i-й элемент результата = f(xs[i]). Размер результата равен размеру xs. Сложность O(n). Пример: map(square, [1, 2, 3]) → [1, 4, 9]. Тип результата: последовательность значений типа f(x) при x типа элемента xs.

**reduce(init, f, xs)** — **свёртка (fold):** начальное значение **init**, на каждом шаге обновляем его: init = f(init, x) для очередного элемента x. После обхода всех элементов получаем одно итоговое значение. Сложность O(n). Примеры: reduce(0, add, xs) — сумма; reduce(1, mul, xs) — произведение; reduce("", concat, strings) — конкатенация строк. Порядок обхода может быть задан (слева направо или справа налево); для ассоциативной f результат не зависит от порядка (например, сумма). В C++17 добавлен std::reduce для параллельной свёртки без заданного порядка.

**where(pred, xs)** — **фильтр:** оставить только те элементы xs, для которых предикат **pred(x)** истинен. Размер результата ≤ размер xs. Сложность O(n). Пример: where(is_even, [1, 2, 3, 4]) → [2, 4]. В стандартной библиотеке C++ — std::copy_if (копирование в выходной итератор по условию).

**Реализация функций (код):**
```cpp
#include <vector>
#include <iterator>
#include <functional>

// map(f, xs): применить f к каждому элементу, вернуть новый вектор
template<typename InputIt, typename F>
auto map(InputIt first, InputIt last, F f) {
    using ResultType = std::decay_t<decltype(f(*first))>;
    std::vector<ResultType> result;
    result.reserve(std::distance(first, last));
    for (; first != last; ++first)
        result.push_back(f(*first));
    return result;
}
template<typename Container, typename F>
auto map(const Container& xs, F f) {
    return map(std::begin(xs), std::end(xs), std::move(f));
}

// reduce(init, f, xs): свёртка — init = f(init, x) для каждого x (слева направо)
template<typename InputIt, typename T, typename BinaryOp>
T reduce(InputIt first, InputIt last, T init, BinaryOp f) {
    for (; first != last; ++first)
        init = f(std::move(init), *first);
    return init;
}
template<typename Container, typename T, typename BinaryOp>
T reduce(T init, BinaryOp f, const Container& xs) {
    return reduce(std::begin(xs), std::end(xs), std::move(init), std::move(f));
}

// where(pred, xs): оставить только элементы, для которых pred(x) истинно
template<typename InputIt, typename Pred, typename OutputIt>
OutputIt where(InputIt first, InputIt last, Pred pred, OutputIt out) {
    for (; first != last; ++first)
        if (pred(*first))
            *out++ = *first;
    return out;
}
template<typename Container, typename Pred>
auto where(const Container& xs, Pred pred) {
    using T = std::decay_t<decltype(*std::begin(xs))>;
    std::vector<T> result;
    where(std::begin(xs), std::end(xs), pred, std::back_inserter(result));
    return result;
}

// Пример использования
#include <iostream>
int main() {
    std::vector<int> v = {1, 2, 3, 4, 5};
    auto squared = map(v, [](int x) { return x * x; });           // [1,4,9,16,25]
    int sum = reduce(0, [](int a, int b) { return a + b; }, v);    // 15 — reduce(init, f, xs)
    auto evens = where(v, [](int x) { return x % 2 == 0; });       // [2,4]
}
```
В стандартной библиотеке: **std::transform** (map), **std::accumulate** / **std::reduce** (reduce), **std::copy_if** (where с выводом в итератор).

## 1.16. Понятие монады (как идиомы и структуры данных). Пример

**Монада как идиома программирования.** Так же, как у идиомы «указатель» есть разные реализации (сырой указатель, умный указатель, итератор), у нас есть идиома **«монада»**. Формально **монада** M⟨T⟩ — это **параметризованный тип данных**, для которого определены операции:

- **from : T → M⟨T⟩** — **«единица»** (unit): функция, погружающая значение типа T в M⟨T⟩, т.е. порождение монады из экземпляра. В других обозначениях: **return**.
- **flatMap : M⟨T₁⟩ × (T₁ → M⟨T₂⟩) → M⟨T₂⟩** — отображение, которое **распаковывает** значение внутри исходной монады, применяет преобразование (функцию T₁ → M⟨T₂⟩) и возвращает результат в монаде M⟨T₂⟩. В других обозначениях: **bind** (>>=).

Математически **from** — мономорфизм: различным T соответствуют различные M⟨T⟩, но обратного взаимно-однозначного соответствия нет (например, в монаду могут быть инкапсулированы ошибки, несколько T могут отображаться в одно M⟨T⟩).

В результате получается **стандартизованный интерфейс** и набор операций, применимых ко всем типам, реализующим эту идиому (Option, Result, List и т.д.). Независимо от особенностей конкретной реализации, над ней можно выполнять одни и те же операции (from, flatMap). При этом алгоритмическая сложность этих операций может быть разной для разных реализаций.

**Что такое монада (абстракция).** Монада — абстракция для **последовательных вычислений с контекстом**: значение «обёрнуто» в контекст. Операции: **return** (from) — упаковать «голое» значение в монаду; **bind** (flatMap) — применить функцию к значению внутри монады (функция возвращает монаду); результат bind — снова монада. Цепочку вызовов можно строить без разворачивания на каждом шаге.

**Зачем.** Без монады код с Option: «получить x из find(); если нет — выйти; получить y из getAge(x); если нет — выйти; …» — много вложенных if. С bind: find(id).bind(getAge).bind(validate) — если любой шаг даёт None, вся цепочка даёт None; иначе результат проходит по цепочке. Контекст («может не быть») переносится автоматически.

**Законы монады** (должны выполняться для любой разумной реализации):
- **Левая единица:** return(a).bind(f) ≡ f(a).
- **Правая единица:** m.bind(return) ≡ m.
- **Ассоциативность:** m.bind(f).bind(g) ≡ m.bind([&](auto x) { return f(x).bind(g); }).

**Option как монада.** return(x) = Some(x). bind(m, f): если m = Some(x), то f(x) (f возвращает Option); иначе None. Так цепочка вычислений с возможным «отсутствием» записывается без явных проверок на каждом шаге.

**Реализация return и bind для Option (код):**
```cpp
#include <optional>
#include <utility>

// return (from): упаковать значение в optional
template<typename T>
std::optional<T> mreturn(T value) {
    return std::optional<T>(std::move(value));
}

// bind (flatMap): если m содержит значение x, вернуть f(x); иначе пустой optional того же типа, что f(x)
template<typename T, typename F>
auto bind(const std::optional<T>& m, F f) -> decltype(f(std::declval<const T&>())) {
    if (!m.has_value())
        return std::nullopt;
    return f(*m);
}

// Пример: find пользователя по id → получить возраст → проверить возраст ≥ 18
std::optional<int> find_age(int user_id) {
    if (user_id == 1) return 20;
    if (user_id == 2) return 16;
    return std::nullopt;
}
std::optional<bool> check_adult(int age) {
    if (age >= 18) return true;
    return std::nullopt;
}

void example() {
    std::optional<int> user_id = mreturn(1);  // from: погрузить значение в optional
    // Цепочка без монады: вложенные if
    std::optional<bool> result_manual;
    if (user_id) {
        auto age = find_age(*user_id);
        if (age) result_manual = check_adult(*age);
    }
    // С bind: одна цепочка
    auto result = bind(user_id, [](int id) {
        return bind(find_age(id), [](int age) { return check_adult(age); });
    });
    // result = true для user_id=1; nullopt для user_id=2 или отсутствующего id
}
```
**Удобная запись:** в языках с поддержкой (Haskell, Rust, F#) используется синтаксис do-notation / computation expressions; в C++ цепочку можно записать через вложенные лямбды, как выше, или через вспомогательный operator>>= для optional (если определить перегрузку).
